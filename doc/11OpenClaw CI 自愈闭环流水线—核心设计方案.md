# Agent CI 自愈闭环 — 核心设计方案 v5.0（最终版）

## 设计目标

将 Jenkins Pipeline 构建失败改造为 AI 驱动的自愈闭环系统。
核心原则：**安全优先、分层决策、人机协同、反馈闭环**。

---

## 二、七条核心规则

| # | 规则 | 说明 |
|---|------|------|
| ① | **白名单门控** | 非授权 Repo / 主干(main) / 保护分支(release/*) → 直接拦截 |
| ② | **故障预判** | Flaky Test / 基建宕机 / 安全阻断 → 不进 AI, 直接人工 |
| ③ | **分级自愈** | 🟢 编译/依赖/配置 → 自动修; 🟡 测试/逻辑 → Patch人工确认; 🔴 业务代码 → 禁止 |
| ④ | **决策矩阵** | 错误类型 × Git权限 × 置信度 三元组决定处理路径 |
| ⑤ | **强化重试** | 最多3次，每次携带历史Reward + 扩大上下文 + 换更强模型 |
| ⑥ | **熔断保护** | 重试超限 → 停止自愈 → 分级告警 → 进入冷却期 → 人工兜底 |
| ⑦ | **反馈闭环** | PR审核结果入库 → 模型微调 → Prompt迭代 → 质量持续提升 |

## 三、模块化划分（可独立开发 + 可扩展）

```
ci-selfheal/
│
├── gate/                          ← G0+G1 模块
│   ├── whitelist.py               # 白名单校验 (可配置)
│   └── precheck.py                # 故障预判规则 (可扩展)
│
├── collector/                     ← S1 模块
│   ├── jenkins_logs.py            # 日志拉取
│   ├── diff_parser.py             # 源码差异
│   └── desensitizer.py            # 脱敏处理
│
├── diagnoser/                     ← S2 模块
│   ├── prompt_builder.py          # Prompt 组装
│   └── ai_client.py               # AI 调用 (可切换模型/Provider)
│
├── decider/                       ← S3 模块
│   ├── matrix.py                  # 决策矩阵表 (可配置)
│   ├── git_branch.py              # Git 操作
│   └── patch_writer.py            # Patch/报告 生成
│
├── verifier/                      ← S4 模块
│   ├── ci_trigger.py              # 触发重建
│   ├── pr_manager.py              # PR 创建 + 标签
│   └── feedback.py                # 结果入库 + Reward 更新
│
├── infra/                         ← 底座
│   ├── lock.py                    # 并发控制 (文件锁)
│   ├── audit.py                   # 审计日志
│   └── notify.py                  # 通知推送 (Slack/钉钉/邮件)
│
├── orchestrator.py                # 主编排器, 串联所有模块
└── webhook_listener.py            # HTTP 入口, 接收 CI 通知
```

## 四、决策矩阵速查表

| 错误类型 | 有Git+权限 | 无权限/无Git |
|----------|-----------|-------------|
| 编译/依赖/配置 | 🔧 自动分支 → Push → 重建 → PR | 📄 Patch文件 → 人工应用 |
| 测试/逻辑 | ⚠️ Patch → 通知维护者(不自动Push) | ⚠️ Patch → 通知运维 |
| 环境/基础设施 | 🔧 建议报告 → 通知运维 | 🔧 建议报告 → 通知运维 |
| CANNOT_FIX_SRC | 🛑 终止 → 通知人工 | 🛑 终止 → 通知人工 |
| 置信度 < 0.60 | ⬇ 降级为诊断报告 | ⬇ 降级为诊断报告 |

## 五、扩展点（可插拔）

| 扩展点 | 位置 | 说明 |
|--------|------|------|
| 新增 CI 引擎 | `collector/` | 加 `gitlab_ci.py` / `github_actions.py` |
| 新增故障预判规则 | `gate/precheck.py` | 加一条 `elif` 规则即可 |
| 切换 AI 模型 | `diagnoser/ai_client.py` | 改 `--model` 参数 |
| 新增通知渠道 | `infra/notify.py` | 加 `wechat()` / `sms()` |
| 调整决策矩阵 | `decider/matrix.py` | 改 YAML/JSON 配置文件 |
| 白名单管理 | `gate/whitelist.py` | 读 YAML 配置, 动态加载 |

# DevOpsClaw

> **全球首创 pipecircle** - AI 驱动的 CI/CD 闭环修复系统
> 
> **新范式 CI/CD 系统**：高稳定高可用 Jenkins 引擎 + 新质生产力 OpenClaw AI 驱动

---

## 快速开始

### 一键部署

```bash
cd DevOpsClaw
cp .env.example .env
chmod +x deploy_all.sh
sudo ./deploy_all.sh
```

选择部署模式：
- **[1] 完整部署 + Nginx**（生产环境）
- **[4] 核心部署**（本地开发，推荐）

### Docker Compose 方式

```bash
cp .env.example .env
docker compose up -d
```

### 自愈闭环

```
┌─────────────────────────────────────────────────────────────────────┐
│                       CI 构建失败触发                                │
│                 Jenkins / GitLab CI / GitHub Actions                 │
└───────────────────────────┬─────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────┐
│  G0 · 安全门控                                                       │
│  · 白名单校验: 仓库 + 分支是否允许自愈?                                │
│  · 保护分支(main/release/*) → 直接拦截                                │
│  · 去重锁: 同一 (repo + 分支 + 错误类型) 不重复触发                    │
└─────────────────┬───────────────┬───────────────────────────────────┘
                  │               │
           ✅ 通过          ❌ 拦截 → 通知 + 记录
                  │
                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  S1 · 信息收集                                                       │
│  · 构建日志(raw + parsed)                                           │
│  · Pipeline DSL / JJB YAML（只读，不修改）                            │
│  · 源码差异(diff) + 环境快照                                         │
│  · 历史失败模式匹配 + 脱敏处理                                       │
└───────────────────────────┬─────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────┐
│  G1 · 故障预判                                                       │
│  直接拦截不可自愈场景:                                                │
│  · 基础设施宕机 / Flaky Test / 安全阻断 / 第三方服务不可用             │
│  · 业务逻辑 Bug（断言失败 / 功能回归）→ 强制人工                       │
│  · 可自愈: 编译错误 / 依赖冲突 / 配置语法 / 环境变量缺失                │
└─────────────────┬───────────────┬───────────────────────────────────┘
                  │               │
           ✅ 可自愈         ❌ 不可自愈 → 生成诊断报告 → 通知人工
                  │
                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  S2 · AI 诊断                                                        │
│  输入: 脱敏后的日志 + DSL + diff + 历史故障                           │
│  输出: 根因定位 + 修复代码 + 类型标签 + 置信度(0.0~1.0)               │
│    · ≥ 0.85 → 高置信, 可自动                                         │
│    · 0.60~0.85 → 中置信, 需验证                                      │
│    · < 0.60 → 低置信, 降级为仅建议                                    │
└───────────────────────────┬─────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────┐
│  S3 · 修复决策                                                       │
│                                                                      │
│  决策矩阵 (错误类型 × Git权限 × 置信度):                              │
│  ┌─────────────────┬──────────────────────┬─────────────────────┐    │
│  │  错误类型        │  ✅ 有Git + 有权限    │  ❌ 无权限/无Git      │    │
│  ├─────────────────┼──────────────────────┼─────────────────────┤    │
│  │ 编译/依赖/配置   │  开fix分支→commit→push │  生成Patch文件       │    │
│  │                 │  → 触发重建 → 提PR     │  供人工审核后手动应用 │    │
│  ├─────────────────┼──────────────────────┼─────────────────────┤    │
│  │ 测试失败/逻辑变更│  仅生成Patch          │  生成Patch           │    │
│  │                 │  ← 通知维护者, 不重建   │  ← 通知运维          │    │
│  ├─────────────────┼──────────────────────┼─────────────────────┤    │
│  │ 环境依赖/端口/   │  生成修复建议          │  生成修复建议        │    │
│  │ 基础设施配置     │  ← 通知运维           │  ← 通知运维           │    │
│  └─────────────────┴──────────────────────┴─────────────────────┘    │
│                                                                      │
│  特殊: CANNOT_FIX_SRC (需改业务代码) → 直接终止, 通知人工              │
│  置信度 < 0.60 → 无论类型, 降级为诊断报告                              │
└─────────────────────────────────────────────────────────────────────┘
                            │
                  ┌─────────┴─────────┐
                  │                   │
           触发自动重建          Patch/报告输出
            (仅分支1)              → 人工审核
                  │
                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  S4 · 验证闭环                                                       │
│                                                                      │
│  自动重建路径:                                                        │
│    ✅ 成功 → 自动提PR(fix→target) + AI诊断说明 → 人工Review            │
│    ❌ 失败 → 重试判断(最多3次) → 每次携带历史Reward信号                 │
│              → 超限 → 熔断 → 分级告警 → 人工兜底                      │
│                                                                      │
│  人工路径:                                                            │
│    → 人工审核Patch → 手动应用 → 触发重建 → 记录结果                    │
│                                                                      │
│  反馈闭环:                                                            │
│    PR结果(accepted/rejected/modified) → 入库 → Prompt优化             │
└─────────────────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════════
  [ 底座 ] · 贯穿所有阶段的基础能力
  · 🔒 并发控制: 同一仓库+分支 仅处理一个事件（文件锁/信号量）
  · 📝 审计日志: 所有AI决策+自动操作 全链路可追溯
  · 🔌 扩展点: G0白名单/G1预判规则/S2模型/S3决策矩阵 均可配置化
═══════════════════════════════════════════════════════════════════════
```

## 常见问题

### Q: Docker Compose 安装失败

**如果使用 Docker Desktop（WSL 集成）：**
1. Settings → Resources → WSL Integration
2. 启用你的发行版
3. 重启 WSL: `wsl --shutdown`

**手动安装：**
```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list
sudo apt-get update && sudo apt-get install docker-compose-plugin
```
